1 透過 USB flash drive 來安裝 CentOS (適用 CentOS 6.5+) [2014-03-09 日]

找一台有安裝 Linux 作業系統的電腦, 下載 CentOS DVD 映像檔1. 你只須 DVD1 就能完成安裝, DVD2 內只包含附加的 RPM，你可以按需要在日後安裝它們。 安裝程式在安裝時不會徵用 DVD22.

在此我們假設下載的版本是 CentOS x86_64. 下載完後 cd 到下載目錄.

接著, 插入 USB flash drive (建議用 USB3.0 同時讀速最好有 30MB +), 並用指令 fdisk 來確認 USB flash drive 位置. 

1: [root@centos ~]# fdisk -l

在此假設查詢到的位置是 /dev/sdb

最後執行下列指令 3 來製作 bootable USB flash drive.

1: [root@centos ~]# dd if=CentOS-6.5-x86_64-bin-DVD1.iso of=/dev/sdb

其作業時間視 USB flash drive 的寫速決定.

完成就可以用該 USB flash drive 來安裝 CentOS. 安裝前記得調整開機裝置順序, 選取 hard disk (or USB HDD) 為第一順位. 進入安裝後, 當詢問資料來源時, 指向對應的 USB flash drive 即可.

2 Third Party Repositories(第3方套件庫) [2014-03-16 日]

2.1 Installing

為了取得更多的軟體, CentOS 要安裝額外的 package. 這裡只是記錄安裝和移除的方法, 其他的設定之後用到在紀錄. 另外, 可以參考 這裡 來取得更多的 repository.
注意! 請先行安裝 RPMforge 再安裝 EPEL 4

2.1.1 確認目前 CentOS 版本與 CPU 架構

1: [root@centos ~]# cat /etc/redhat-release
2: CentOS release 6.5(Final)
3: 
4: [root@centos ~]# uname -i
5: i386

2.1.2 Installing RPMforge

  1. Download the rpmforge-release package 5. (下載 RPMForge 最新 release 的 RPM 檔案) 
    1: $ cd /etc/yum.repos.d
2: $ sudo wget http://pkgs.repoforge.org/rpmforge-release/rpmforge-release-0.5.3-1.el6.rf.i686.rpm
  2. Install RPMforge's GPG key. (安裝 RPMforge 釋出的數位簽章) 
    1: $ sudo rpm --import http://apt.sw.be/RPM-GPG-KEY.dag.txt

    If you get an error message like the following the key has already been imported. 

    error: http//apt.sw.be/RPM-GPG-KEY.dag.txt: key l import failed.
  3. Verify the package you have downloaded. (驗證 RPM 檔案的簽章資訊是否與系統內的簽章相符(前一步安裝的簽章)) 
    1: $ sudo rpm -K rpmforge-release-0.5.3-1.el6.rf.i686.rpm
  4. Install the package 
    1: $ sudo rpm -i rpmforge-release-0.5.3-1.el6.rf.i686.rpm
  5. 完成後, 可以用下列指令來檢查是否安裝成功 
    1: $ sudo yum repolist

    如果有看到 rpmforge 就代表成功了 :)

2.1.3 Installing EPEL(Extra Packages for Enterprise Linux)

  1. Download the epel-release package 6 . 
    1: $ cd /etc/yum.repos.d
2: $ sudo wget http://mirror01.idc.hinet.net/EPEL/6/i386/epel-release-6-8.noarch.rpm
  2. Installing EPEL's GPG key. 
    1: $ sudo rpm --import https://fedoraproject.org/static/0608B895.txt
  3. Veryfy the package you have downloaded. 
    1: $ sudo rpm -K epel-release-6-8.noarch.rpm
  4. Install the package. 
    1: $ sudo rpm -i epel-release-6-8.noarch.rpm
  5. 完成後, 可以用下列指令來檢查是否安裝成功 
    1: $ sudo yum repolist

    如果有看到 epel 就代表成功了 :)

3 Mounting an NTFS filesystem on CentOS [2014-03-16 日]

3.1 Installing required packages

For CentOS-6 the EPEL repository is carrying later NTFS packages. To install EPEL(-> 2.1.3 Installing EPEL)

Installing ntfs-3g 

1: $ sudo yum install ntfs-3g

You may also want to 

1: $ sudo yum install ntfsprogs ntfsprogs-gnomevfs

for additional functionality.

3.2 Mounting an NTFS filesystem

Using fdisk to find your nfts filesystem 

1: $ fdisk -l

Suppose ntfs filesystem is /dev/sdc and you are going to mount it on /mnt/win, do the following

First, create a mount point 

$ sudo mkdir -p /mnt/win

Next, edit /etc/fstab as follows.

To mount read-only: 

$ sudo /dev/sdc /mnt/win ntfs-3g ro,umask=0222,defaults 0 0

To mount read-write: 

$ sudo /dev/sdc /mnt/win ntfs-3g rw,umask=0000,defaults 0 0

You can now mount it by running: 

$ sudo mount /mnt/win

5 安裝 Transmission [2014-03-29 六]

Website: http://www.transmissionbt.com/
version: 2.76

5.1 安裝

連接到 Transmission 官網, 點擊 CentOS 圖示下的 Contrlb Packages 連結到另一個網站 Geekery, 網站提供了現成的 repo 檔, 可以讓我們方便的安裝 Transmission.

所以, 先下載 Greekery 提供的 repo 檔案. 

1: $ cd /etc/yum.repos.d/
2: $ sudo wget http://geekery.altervista.org/geekery-el6-i686.repo

接著, 安裝 Transmission 

1: $ sudo yum install transmission

當 yum 詢問 GPG key 時, 輸入 y 

1: warning: rpmts_HdrFromFdno: Header V3 DSA signature: NOKEY, key ID a4673575
2: geekery/gpgkey                                                                                                                   | 1.3 kB     00:00
3: Importing GPG key 0xA4673575 "Marco Spallacci <marco.spallacci@gmail.com>" from http://geekery.altervista.org/download.php?filename=GEEKERY-GPG-KEY
4: Is this ok [y/N]: y

雖然不是很了解, 但這和之前安裝 RPMforge 的方式不同. RPMforge 的安裝是下載 rpm 檔與 GPG key 後, 驗證再安裝. 完成後會在 /etc/yum.repos.d 下出現 RPMforge 的 repo 檔. 而這裡的方式卻直接下載 repo 檔, 有時間應該要搞清楚 rpm 與 repo 的關係.

啟動 Transmission, 自動產生設定檔 (settings.json)

1: $ sudo service transmission-daemon start

PS.

  1. daemon 代表提供服務的程式.
  2. service 用來啟動 daemon 的 script

5.2 設定

5.2.1 Creating Download and Incomplete Directory

建立完成檔案資料夾並指定使用者與群組 

$ sudo mkdir -p /home/transmission/Downloads
$ sudo chown transmission:transmission /home/transmission/Downloads

建立未完成檔案資料夾並使用者與群組 

$ sudo mkdir -p /home/transmission/Incompletes
$ suod chown transmission:transmission /home/transmission/Incompletes

在這裡要設定使用者與群組的原因在於 Transmission 的下載檔案使用者與群組皆為 transmission, 如果不指定為 transmission, 檔案會因為權限無法存放在指定的資料夾.

5.2.3 Editing Firewall

關閉 IPv4 防火牆, 以 vi 開啟 IPv4 防火牆設定檔 

$ sudo service iptables stop
$ sudo vi /etc/sysconfig/iptables

在設定檔中開啟 Transmission 的 port , 下列新增的 port 都是 Transmission 的 預設值. 

-A INPUT -p tcp -m tcp --dport 9091 -j ACCEPT
-A INPUT -p tcp -m tcp --dport 51413 -j ACCEPT
-A INPUT -p tcp -m tcp --dport 49153:65534 -j ACCEPT

存檔退出並重新開啟 IPv4 防火牆 

$ sudo service iptables start

IPv6 的防火牆也可同時設定, 步驟與參數跟 IPv4 相同 

$ sudo service ip6tables stop
$ sudo vi /etc/sysconfig/ip6tables

在設定檔中加入下列參數(-> 5.2.3 Transmission:ipv4設定)後, 存檔並啟動 IPv6 防火牆 

$ sudo service ip6tables start

5.3 啟動與關閉 Transmission

5.3.1 啟動 Trnasmission

$ sudo service trasnmission-daemon start

5.3.2 關閉 Transmission

$ sudo service transmission-daemon stop
$ sudo killall transmission-daemon

5.3.3 重新啟動 Transmission

$ sudo service transmission-daemon restart

5.4 登入

遠端登入是用瀏覽器網址輸入 IP:9091 (or 域名:9091), 本機則是 localhost:9091. 如果是使用 DHCP 上網, 則可以登入進設備(Ex: 無線AP), 讓系統每次拿到的 IP 都是同一組 :)

6 安裝 JDownloader [2014-03-30 日]

Website: http://jdownloader.org/
version: 0.9.581

首先, 建立先用 wget 下載官網提供的腳本檔案(js.sh), 存放的地方任你選. 

$ wget -c http://installer.jdownloader.org/jd_unix_0_9.sh

賦予腳本檔執行權限 

$ chomd +x jd.sh

執行腳本檔. 執行後會開始安裝程式，裝完就可以使用了 :) 

$ start jd.sh
